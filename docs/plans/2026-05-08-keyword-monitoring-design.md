# 关键词监控共识记录

本文档记录“事件话题 / 关键词监控系统”中，当前已经确认的关键词抽取共识。

## 目标

关键词用于观察资讯中的新信号和热度变化。

关键词不是文章分类、摘要标签或泛化主题，而是文章事件中的核心主体名词。

## 抽取对象

关键词通常应是文章事件里的主体、对象或核心名词，包括：

- 公司、机构、组织
- 产品、工具、平台
- 模型、代号、项目名
- 人名
- 政策、法规、标准
- 技术名词
- 短概念词

关键词应优先来自原文中明确出现的名词或名词短语。

## 不应抽取

不应抽取泛化类别词、标题式短语或事件句。

无效方向示例：

- AI
- 人工智能
- 科技
- 商业
- 模型
- 产品
- 行业趋势
- 产品更新
- OpenAI 推出成人模式
- 企业加速采用 AI

## 数量限制

每篇进入关键词流程的文章输出 1-3 个关键词。

抽取时宁可少，不要为了凑满数量而输出弱相关或泛化词。

## 长度限制

每个关键词长度不超过 20 个字符。

字符按文本长度计算，英文、数字、空格、连字符和标点都计入长度。

有效示例：

- OpenAI
- GPT-5.1
- Claude Code
- ChatGPT
- Agent SDK
- AI Act
- Microsoft Copilot
- 年龄识别
- 成人模式
- AI浏览器
- 具身智能

超过长度或过于短句化的关键词应被缩短、替换为更小的核心名词，或丢弃。

## 类型

关键词第一版直接带类型。

类型枚举：

- `org`：公司、机构、政府部门、大学、实验室、组织
- `person`：人物
- `product`：产品、工具、平台、应用、功能
- `model`：AI 模型、模型系列、模型版本
- `technology`：技术、协议、框架、API、数据集、基准、算法方法
- `hardware`：芯片、机器人、设备、算力基础设施、数据中心
- `policy`：政策、法规、标准、监管框架
- `case`：具体案例、案件、交易、收购、融资、事故、项目、战事、会议
- `topic`：可追踪的短议题、应用方向、行业现象

`case` 用于名词化的具体案例或事件对象，但必须保持短名词短语，不得输出完整新闻标题。

## 别名与归一

关键词表需要包含别名字段（`归一项`）。

每次写入关键词前，先查询关键词表：

- 如果抽取名称经 NFKC + lower 后命中已有关键词的规范名称，使用该规范名称。
- 如果抽取名称经 NFKC + lower 后命中已有关键词的别名，转换为该记录的规范名称。
- 如果没有命中，则创建新的关键词记录。

`归一项` 只存真实别名或人工维护别名，不存规范名自身的小写结果。大小写、全半角等机械归一由程序内存索引处理。

热度统计和后续监控按规范名称聚合，不按每次抽取到的原始名称分散统计。

## 新闻表关联

新闻表和过滤表保留 `关键词` 多选字段作为原始抽取结果，并新增 `关键词记录` 关联字段，用于关联关键词表。

每篇新闻写入后，将 1-3 个归一后的关键词关联到该新闻记录。

关键词表只维护关键词自身信息、别名、类型和统计字段，不在关键词表里堆新闻 ID 列表。
